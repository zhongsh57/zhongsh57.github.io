响应式布局的宽度自适应

我们的容器的高度 如果不对高度进行设置的话 那么默认的高度就是 content 的高度
那么宽度呢？ 因为 blok 和 inline-block 的存在 宽度可以是无限延伸的 那么宽度的自动适应就需要使用我们的一个很少使用的值了

最大宽度就是 不可以断行的最大元素的宽度

多个元素的时候对兄弟元素的样式进行设置

有一个特殊选择器

这里总结一下我们的伪类选择器

选择器 作用
li:first-child 选择属于当前父元素的 li 子元素的第一个
li:nth:child(odd) 选择每奇数个子元素
li:nth:last-child(odd) 倒数第几个元素

这里要注意 是没有正数第几个元素的

还有我们的 nth 标签 这种寻择器我们之前都理解错了！！！！！！
我们理解的是 当前的标签下面的我们选择的标签 如果是对应的标签那么就加上对应的样式 如果不是 那么就不会起作用

如果用上了 nth-child(odd) 那么奇数个就会被加上样式 也就是说 他会选择当前的直接父元素下面的选定的第几个元素 然后加上对应的样式 这个伪类选择器还可以叠加使用 下面来点例子

前置选择器

span ~ li
这个选择器 会选择前面有 span 标签的所有 li 标签

这里面的隐藏的特性也有范围选择器比如说 li:nth-child(n+4) 这个就是会选择第四个元素以后的 包括第四个元素 然后给他们加上样式 这个数字的计数 跟我们上面的一样 也会选择当前的子元素 然后如果不符合 那么就不会加上样式 这里还有一点就是嵌套的元素 那么里面的元素会在倒数的第一个 比如说 div 里面有一个 li 那么 li 的 last 1 就是里面的 li 而不是外面的 div

这个语法糖的本质就是 括号里面原来只接受的是数字 然后如果 n+4 n 的取值就会从 0 开始逐项递增 相当于

具体的最大值我们也不知道 反正也不可能有这么多数字

垂直居中 定位 absolute 和 relative 的坑

我们常说的定位 并不是以我们的元素的中心定位的 而是以元素的左上角定位的
这个时候 如果一个盒子有高度和宽度的话 那么 设置定位是 50% 50% 那么就是这个盒子的左上角在这个地方 并不是盒子的中间在这个地方

keyframe的用法

每一个不同的frame就是不同状态下面对应的css状态
比如说
@keyframes bounce {
60%,
80%,
to {
transform: translateY(350px);
animation-timing-function: ease-out;
}
70% {
transform: translateY(250px);
}
90% {
transform: translateY(300px);
}
}
也就是从一开始 的不同的状态 然后经过animation算法 就可以得到平滑的动画
这里面的比较难以理解的就是 里面的to 就是100%的缩写 from就是0% 的缩写
